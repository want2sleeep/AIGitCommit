# 快速开始指南

## ✅ 插件已安装成功！

插件 `AI Git Commit` 已经安装到你的 VSCode 中。

---

## 🚀 开始试用

### 步骤 1: 重启 VSCode（推荐）

虽然不是必需的，但重启 VSCode 可以确保插件完全加载：

1. 关闭所有 VSCode 窗口
2. 重新打开 VSCode

或者按 `Ctrl+Shift+P`，输入 "Reload Window" 重新加载窗口。

---

### 步骤 2: 配置 API

#### 方法 1: 使用配置向导（推荐）

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "配置 AI Git Commit"
3. 按照提示输入：
   - **API 端点**: 例如 `https://api.openai.com/v1`
   - **API 密钥**: 你的 OpenAI API 密钥
   - **模型名称**: 例如 `gpt-3.5-turbo`

#### 方法 2: 手动配置

1. 按 `Ctrl+,` 打开设置
2. 搜索 "AI Git Commit"
3. 配置以下选项：
   - `aigitcommit.apiEndpoint`: API 端点
   - `aigitcommit.modelName`: 模型名称
4. 运行 "配置 AI Git Commit" 命令设置 API 密钥

#### 测试配置示例

**使用 OpenAI:**
```
API 端点: https://api.openai.com/v1
模型名称: gpt-3.5-turbo
API 密钥: sk-your-api-key-here
```

**使用 Ollama (本地):**
```
API 端点: http://localhost:11434/v1
模型名称: llama2
API 密钥: (任意值或留空)
```

---

### 步骤 3: 测试生成功能

#### 准备测试环境

1. **打开一个 Git 项目**（或者就用当前项目）
2. **修改一个文件**，例如：

```typescript
// 在任意文件中添加一个简单的函数
export function testFunction() {
  return "Hello, AI Git Commit!";
}
```

3. **暂存变更**：
   - 打开源代码管理视图 (`Ctrl+Shift+G`)
   - 点击文件旁边的 `+` 号暂存变更
   - 或者在终端运行: `git add .`

#### 方法 1: 使用 SCM 视图按钮（最简单）

1. 打开源代码管理视图 (`Ctrl+Shift+G`)
2. 在标题栏找到 ✨ 图标按钮
3. 点击按钮
4. 等待 AI 生成提交信息
5. 查看生成的结果

#### 方法 2: 使用命令面板

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 "生成AI提交信息"
3. 按回车执行
4. 等待生成完成

#### 方法 3: 使用快捷键

1. 确保在 Git 项目中
2. 按 `Ctrl+Shift+G, C`
3. 等待生成完成

---

### 步骤 4: 查看和编辑结果

生成完成后，你会看到：

1. **提交信息输入框**显示生成的内容
2. 可以**直接编辑**生成的提交信息
3. 可以点击**重新生成**获取新的提交信息
4. 可以点击**取消**放弃本次操作
5. 满意后直接**提交**

## ⚙️ 调整设置

### 常用设置

打开设置 (`Ctrl+,`)，搜索 "AI Git Commit"：

- **语言**: 
  - `zh-CN` (中文)
  - `en-US` (英文)

- **提交格式**:
  - `conventional` (约定式提交，推荐)
  - `simple` (简单格式)

- **最大 Token 数**: 
  - 默认 500
  - 可以调整为 300-1000

- **温度参数**:
  - 默认 0.7
  - 范围 0-2
  - 越低越保守，越高越有创意

## 🐛 常见问题

### 问题 1: 提示"无暂存变更"

**原因**: 没有暂存任何文件

**解决**:
1. 修改一些文件
2. 在源代码管理视图中暂存变更
3. 或运行 `git add .`

### 问题 2: API 调用失败

**原因**: API 配置错误或网络问题

**解决**:
1. 检查 API 端点是否正确
2. 检查 API 密钥是否有效
3. 检查网络连接
4. 查看输出日志了解详细错误

### 问题 3: 找不到命令

**原因**: 插件未正确加载

**解决**:
1. 重启 VSCode
2. 确认插件已启用
3. 确认在 Git 项目中（插件只在 Git 仓库中激活）

## 📊 性能提示

### 优化生成速度

1. **使用更快的模型**: 
   - GPT-3.5-turbo 比 GPT-4 快
   - 本地模型（Ollama、vLLM）最快

2. **减少 diff 大小**:
   - 分批提交变更
   - 避免一次暂存太多文件

3. **调整 max_tokens**:
   - 减少到 300-400 可以加快响应
